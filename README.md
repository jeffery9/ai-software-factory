# 🎓 Factory Meta Builder (L0)

> **The Architect of Architects — Designing the Future of Software Factories.**

Factory Meta Builder is the **L0 (Level Zero)** layer of the AI Software Factory ecosystem. It functions as the "Teacher and Enlightener," providing the thinking models, architectural standards, and evolutionary strategies required to build and maintain high-quality **L1 Builders**.

---

## 🏛️ The Four-Layer Hierarchy

1.  **L0 Meta Builder (This Project)**: The Teacher. Designs the Knowledge Framework and FDL (Factory Definition Language) standards.
2.  **L1 Builder (Architect)**: The Student (e.g., `odoo-factory-builder`). Uses L0 blueprints to construct and evolve L2 Factories.
3.  **L2 Factory (Engine)**: The Working System (Expert Systems + Orchestrators). Executes the actual software engineering tasks.
4.  **L3 Software**: The Final Product (Applications, Modules, APIs).

## 🚀 Key Features

- **FDL (Factory Definition Language)**: Standardized JSON/YAML schemas for defining factory blueprints.
- **Pedagogical Protocol**: A "Teacher-Student" interaction model that emphasizes teaching *how* to solve problems rather than just providing the solution.
- **Curriculum Evolution**: Continuous improvement of architectural standards (SOLID, CQRS, Hexagonal) via the `evolution-expert`.
- **Domain Blueprints**: Pre-defined templates for various technology stacks (Odoo, Go Microservices, Python APIs).

## 🛠️ Specialized Experts (AGENTS.md)

This project employs a "Thin Agent + Granular Experts" architecture. Refer to [AGENTS.md](./AGENTS.md) for the full roster:

- **`meta-builder`**: Oversees the educational strategy and core philosophy.
- **`engine-driver`**: Responsible for drafting FDL blueprints and providing code templates.
- **`evolution-expert`**: Manages the "Knowledge Base" (Curriculum) and updates standards.

## 📐 Pedagogical Protocol (Our "Mantra")

- **Teach to Fish**: We provide the **Protocol** and **Strategy**, not just the code.
- **Chain of Thought**: We guide L1 Builders to think about architectural conflicts and documentation discovery.
- **Evaluation**: We define the success criteria for the evolution of L1 and L2 layers.
